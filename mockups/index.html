Stochastic Modelling — Display Options

Three design patterns for overlaying a range of Monte Carlo outcomes onto the existing dashboard. Each uses identical assumed inputs (age 28→65, 7% mean return, 12% annual volatility, 1,000 simulations) so the same data is displayed in three very different ways. The deterministic projection is always shown as the median or central estimate.

Option A

Fan Chart — Net Worth Percentile Bands

Replace (or toggle) the current stacked-area chart with a net-worth fan chart. The existing breakdown by pot type (pension / ISA / GIA) becomes a separate Details panel rather than the primary chart. Percentile envelopes (10th–90th and 25th–75th) are drawn as shaded bands around the median line.

The deterministic line stays as a dashed overlay so users can see how the simple projection compares to the median of the simulation distribution.

Strengths

  • Immediately communicates uncertainty width
  • Familiar to users of financial planning software
  • The "cone" opening toward retirement is intuitive
  • Shortfall floor is easy to draw as a reference line

Trade-offs

  • Loses the per-pot breakdown (must be a separate view)
  • Crowded if debt bands are also shown below the axis
  • Wide bands in early years can obscure the deterministic line
  • 1,000 simulations × 60 years = 60k data points to aggregate

Option B

Companion Survival Panel — Probability of Success

Keep the existing chart exactly as-is. Add a second narrow chart below it showing, for each age during retirement, the fraction of simulation paths that have not yet run out of money. This is the "probability of success" curve familiar from safe-withdrawal-rate research.

Because it is a separate panel it adds zero visual noise to the existing breakdown chart. The stat card area can additionally surface three headline numbers: P(success to maxAge), median age at exhaustion (for paths that do fail), and the median residual wealth at maxAge.

Strengths

  • Zero changes to the primary chart users already know
  • Single interpretable number: "87% chance of lasting to 90"
  • Works with any spending level — no per-pot modelling needed
  • Easy to add as a toggle without rebuilding anything

Trade-offs

  • Doesn't show magnitude of wealth range, only binary success/fail
  • Two separate charts to read and reconcile
  • Success/fail threshold is blunt (zero wealth = fail)
  • Vertical space is already constrained on smaller screens

Option C

Outcome Distributions at Snapshot Ages

Keep the existing chart as the primary view. Add a row of small horizontal distribution charts ("small multiples") pinned to key ages — retirement, age 75, age 85, and maxAge. Each shows the full distribution of total net-worth outcomes at that age as a smoothed density curve, with the deterministic projection marked as a vertical line.

This answers the question users actually care about: "at age 80, what's the realistic spread of outcomes?" without replacing the main chart or requiring them to reason about a fan shape evolving over time.

Strengths

  • Shows full magnitude of uncertainty, not just percentile lines
  • Deterministic vs stochastic comparison is direct and readable
  • Small multiples are compact — fits in the existing layout
  • Users can pick the age that matters most to them

Trade-offs

  • Doesn't show how paths evolve over time (static snapshots)
  • Four mini-charts adds implementation complexity
  • Density curves require kernel smoothing — more CPU per recalc
  • Limited vertical space on mobile
